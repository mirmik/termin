# Slang Bind-By-Name Runtime Plan

Date: 2026-06-09
Updated: 2026-06-11

## Status

This plan is still active, but it has been refined by the scope-first decision
in `2026-06-11-slang-scope-first-binding.md`.

Bind-by-name is the public runtime API. Scope metadata is the missing piece
that prevents bind-by-name from becoming a thin wrapper around another flat
numeric binding table.

## Goal

Move Termin's shader resource binding model from backend numeric slots to
Slang/reflection-driven symbolic bindings.

The target state is:

- Slang source files do not contain backend-specific layout attributes such as
  `[[vk::...]]` or `register(...)`.
- Shader stage IO is expressed through Slang/HLSL semantics:
  `POSITION`, `TEXCOORD0`, `COLOR0`, `SV_Position`, `SV_Target0`, etc.
- Shader resources are expressed by logical names:
  `per_frame`, `material`, `draw_data`, `u_input`, `albedo_texture`, etc.
- Each resource has a scope: `frame`, `pass`, `material`, `draw`, or
  `transient`.
- Generated artifacts carry enough layout metadata for each backend to bind
  resources correctly.
- Render passes and materials bind resources by logical names, not hard-coded
  integers like `4`, `8`, or `24`.

## Source Model

Slang source should describe logical shader interfaces:

```slang
import termin_prelude;

struct FragmentInput
{
    float2 uv : TEXCOORD0;
};

struct FragmentOutput
{
    float4 color : SV_Target0;
};

struct TonemapParams
{
    float exposure;
    int method;
};

[[TerminScope("pass")]]
ConstantBuffer<TonemapParams> u_params;

[[TerminScope("transient")]]
Sampler2D u_input;

[shader("fragment")]
FragmentOutput main(FragmentInput input)
{
    FragmentOutput output;
    float3 color = u_input.Sample(input.uv).rgb * u_params.exposure;
    output.color = float4(color, 1.0);
    return output;
}
```

The source should not contain Vulkan or D3D layout syntax. Backend placement is
generated by the artifact/layout pipeline. Explicit scope should come from the
Termin Slang prelude via `[[TerminScope("frame")]]`,
`[[TerminScope("pass")]]`, `[[TerminScope("material")]]`,
`[[TerminScope("draw")]]`, or `[[TerminScope("transient")]]`.

`[[termin::scope("frame")]]` is not the target spelling. A real `slangc
2026.1-52-gc8ddf20bb` probe treated `[termin::Scope("frame")]` as an unknown
attribute and omitted it from reflection JSON. Plain user attributes such as
`[[TerminScope("frame")]]` are reflected under `userAttribs`.

## Layout Metadata

Every generated shader artifact should have a sidecar layout file:

```text
<artifact>.layout.json
```

The runtime package shape may also carry aggregate layout files under:

```text
shaders/layout/<uuid>.shader-layout.json
```

Minimum resource fields:

```json
{
  "name": "u_input",
  "kind": "texture",
  "scope": "transient",
  "set": 0,
  "binding": 4,
  "stage_mask": 2,
  "size": 0
}
```

`set` and `binding` are backend placement, not authoring API. The current
Vulkan backend flattens everything to set 0. Future Vulkan scoped sets can map
`scope` to separate descriptor sets without changing pass code.

## Runtime API

Migrated pass code should look like:

```cpp
ctx.bind_uniform("u_params", params_buffer);
ctx.bind_texture("u_input", input_texture);
```

`RenderContext2` resolves the name through the active shader layout. It must
log and skip invalid bindings when:

- the active shader has no layout metadata,
- the resource name is missing,
- the resource kind does not match the bind call,
- backend placement conflicts with the active pipeline layout.

Legacy numeric calls remain available for GLSL and unmigrated paths, but new
Slang paths should not add numeric binding usage.

## Current State

- `termin_shaderc` writes `.layout.json` sidecars for Slang artifacts.
- Runtime shader loading merges sidecar resources into
  `tc_shader_resource_binding[]`.
- Layout sidecars and `tc_shader_resource_binding` now carry resource `scope`;
  old sidecars without that field are accepted through conservative inference.
- `termin_shaderc` consumes `[[TerminScope("...")]]` from Slang reflection
  `userAttribs` and treats invalid values as hard errors. The temporary
  `[[Scope("...")]]` alias is accepted during migration.
- `RenderContext2` has initial `bind_uniform(name)` and `bind_texture(name)`
  methods.
- `RenderContext2` stores pending bindings in scope buckets and flattens them
  for the current backend resource-set API.
- Grayscale, tonemap, and bloom use symbolic resource binding.
- Material/color rendering still has numeric ABI debt for per-frame, shadows,
  material UBO/textures, and draw data.

## Next Tasks

1. Add a Termin Slang prelude with `TerminScopeAttribute` and make generated or
   authored Slang sources import it where explicit scopes are used.
2. Add dirty-per-scope tracking on top of the scoped buckets.
3. Convert `ColorPass` and material UBO/texture upload to bind by name where
   layout metadata exists.
4. Remove generated Slang backend attributes such as `[[vk::push_constant]]`
   from material shader synthesis.
5. Add validation for duplicate backend bindings with incompatible resource
   type/count.
6. Later, map scopes to multiple Vulkan descriptor sets and bind only dirty
   scopes.

## Non-Goals

- Do not require Slang at runtime. Runtime backends consume generated artifacts
  and layout metadata.
- Do not migrate every shader in one step.
- Do not remove legacy numeric APIs until GLSL/material compatibility paths
  have been migrated or retired.
- Do not reintroduce a universal binding table.
