# Архитектура Diffusion Editor

## Назначение

Редактор изображений с упором на рисование нейросетями. Референс — GIMP: послойная работа с изображением, инструменты кисти, маскирование, undo/redo.

## Зависимости

Используются библиотеки экосистемы termin (`termin-base`, `termin-graphics`, `termin-gui`), расположенные рядом в дереве репозитория. Нейросетевые модели хранятся вне проекта — путь к ним указывается в настройках.

---

## Модель документа: послойный стек

Документ — это стек слоёв, образующих дерево. Каждый слой хранит собственные пиксели и участвует в компоузитинге.

Дерево обходится снизу вверх. Каждый слой вносит свой вклад в накопленный результат через alpha-blending с учётом непрозрачности и флага видимости. Слои-группы сначала компоузят своих детей, затем передают результат наверх.

```
А
├─ C
│  ├─ D
│  └─ E
└─ B
F   ← начало
```

### Тайловое хранение

Пиксельные данные разбиты на тайлы фиксированного размера. Это позволяет инвалидировать и перерисовывать только затронутые тайлы, а не весь холст целиком.

### Кеширование и инвалидация

Результат каждого шага компоузитинга кешируется на уровне тайлов. При изменении слоя инвалидируется его кеш и кеш всех вышестоящих слоёв в затронутых тайлах. Структурные изменения (добавление, удаление, перемещение слоёв) сбрасывают кеш полностью.

---

## Типы слоёв

**Paint** — обычный растровый слой. Поддерживает рисование кистью, цветовой пипеткой, ластиком.

**Diffusion** — слой генерации через диффузионную модель. Хранит маску и параметры генерации. Источник изображения — кеш нижележащих слоёв на момент запуска генерации.

**Lama** — слой удаления объектов. Хранит маску зоны удаления.

**Instruct** — слой редактирования по текстовой инструкции (InstructPix2Pix).

Слои с маской (Diffusion, Lama, Instruct) хранят маску и исходный патч рядом с изображением — это нужно для повторного запуска генерации без перерисовки.

---

## Мутации документа

Все изменения состояния документа проходят через единую точку — `DocumentService`. Он принимает команду, выполняет её и регистрирует в истории. Прямое изменение слоёв в обход сервиса недопустимо.

Команды — типизированные структуры данных (не строки, не колбеки). Они описывают намерение явно и могут быть независимо протестированы.

Результаты движков не применяются напрямую. Mapper-слой переводит их в команды документа — это разрывает зависимость между движками и `DocumentService`.

---

## Undo/Redo

История построена на снимках состояния (snapshot-based). При выполнении команды сохраняется состояние до и после. Undo восстанавливает снимок «до», redo — «после».

Для операций рисования (где снимать весь стек накладно) используется патч-подход: в историю записывается только изменённый прямоугольник пикселей.

История ограничена по памяти. При превышении лимита старые записи вытесняются.

В идеале всё, что пользователь может сделать, должно быть покрыто undo/redo.

---

## Движки (ML-модели)

Движок — обёртка над нейросетевой моделью. Общий паттерн для всех движков:

- Модель загружается лениво при первом использовании.
- Инференс выполняется в фоновом потоке.
- Каждый кадр главный цикл опрашивает движок на предмет готового результата.
- При завершении работы движок завершает фоновый поток и освобождает модель.

Текущие движки: диффузия (SDXL), удаление объектов (LaMa), редактирование по инструкции (InstructPix2Pix), сегментация (rembg).

## Слои приложения

**Точка входа** инициализирует окно, OpenGL-контекст, запускает главный цикл и выполняет финальную очистку ресурсов.

**EditorWindow** — оркестратор. Собирает UI, соединяет компоненты через колбеки, направляет события от UI к документу и движкам. Не содержит бизнес-логики.

**Панели** (`BrushPanel`, `DiffusionPanel`, `LayerPanel` и т.д.) — UI-компоненты без прямого доступа к документу. Общаются с оркестратором через колбеки.

**EditorCanvas** — виджет холста. Реализует инструменты рисования и масштабирование. Делегирует изменения документа через колбеки, не трогает стек слоёв напрямую.

**DocumentService** — единая точка применения команд к документу. Всё, что меняет состояние, идёт через него.

**LayerStack** — дерево слоёв с компоузитинговым движком и тайловым кешем. Чистая доменная логика без UI.

**HistoryManager** — стек undo/redo с ограничением по памяти.

**Движки** — изолированные обёртки над ML-моделями. Не знают ни о UI, ни о документе.

---

## Формат проекта

Проект сохраняется в файл-архив. Архив содержит метаданные документа и бинарные данные каждого слоя (изображение, маска, исходный патч). Формат версионирован: при открытии файла новее поддерживаемой версии выдаётся ошибка.

---

## Логирование

Используется общая система логирования из `termin-base`, а не стандартный `print` или `logging`. Уровень логирования задаётся в точке входа.

Исключения нельзя проглатывать молча — любая ошибка должна быть залогирована.
